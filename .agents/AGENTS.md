# セキスペ対策アプリ 開発ルール

## 開発タスク完了時の安全検証ガイドライン

ソースコードの変更、あるいは新規ファイルの追加を伴う実装タスクの完了時には、ユーザーへ完了を報告する前に、エージェントは必ず自律的に以下のチェックを実行してください。

1. **個人情報・機密情報の自動スキャン**
   変更されたファイル、および追加されたファイル内に対して、以下の文字列やパターンが含まれていないかを grep 等で自動スキャンし、結果を検証してください。
   - 開発者のローカルアカウント・ユーザー名（例: `tanaka`）
   - 実在する個人用メールアドレス（例: `@gmail.com` 等）
   - 本物の認証情報（本物のAPIキー、パスワード、秘密鍵、シークレットトークン）が混入していないか

2. **ローカル検証ファイルの除外確認**
   ローカルでのみ動作させる自動テストスクリプト（例: `test_app.py`）や実行キャッシュ（例: `.pytest_cache/`）が、正しく `.gitignore` の定義によって Git 追跡対象（untracked / staged）から除外されているかを確認してください。
